# jv-oop-machine

Завдання: ua.machines.Machine (jv-oop-machine)
Легенда: Нам потрібно керувати різною будівельною технікою. Усі машини вміють починати роботу і закінчувати її, але кожна робить це по-своєму.

Технічне завдання:

Створити абстрактний клас ua.machines.Machine.

У ньому мають бути два абстрактних методи: doWork() та stopWork().

Створити 3 класи, які успадковують ua.machines.Machine:

Truck (Вантажівка)

Bulldozer (Бульдозер)

Excavator (Екскаватор)

У кожному з цих класів перевизначити (override) методи doWork() та stopWork().

Вони мають виводити в консоль повідомлення, специфічні для машини. Наприклад: "Truck started to work", "Bulldozer stopped working".

У класі Main (Application):

Створити масив типу ua.machines.Machine і заповнити його екземплярами трьох різних машин (Truck, Bulldozer, Excavator).

За допомогою циклу for-each (або звичайного for) пройтися по масиву.

У кожній ітерації викликати методи doWork() і stopWork() для поточної машини.
